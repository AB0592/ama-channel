请完整阅读以下项目信息，后续所有开发工作都必须遵守这些约束和规范。

========================================
一、项目简介
========================================

项目名称：阿嬷的频道
定位：专为老年人设计的方言智能点播网站
形态：单文件网页应用（index.html），纯前端，无后端，无数据库
在线地址：https://sagaxm-ops.github.io/ama-channel/
GitHub仓库：https://github.com/sagaxm-ops/ama-channel
部署方式：GitHub Pages（main分支根目录）

目标用户：不会打字、只会说方言（莆仙话、闽南话）的老年人
核心痛点：现有视频平台只能用文字搜索，不支持方言，老人找不到想看的戏曲

========================================
二、已实现的5种点播方式
========================================

1. 浏览器语音输入：点击麦克风，用普通话说出戏名，系统识别并匹配
2. 预录入词条库+模糊匹配：通过"录入莆仙话/闽南话"功能，预先录入习惯叫法，系统模糊匹配
3. 搜索栏文字输入：输入中文、方言文字或不完整戏名，五级候选搜索
4. 听节目名选台（TTS）：系统朗读节目名，长辈听到想看的按"就是这个"确认
5. 浏览海报+二次确认：分类浏览海报，点击后弹出确认卡片

========================================
三、技术栈与架构
========================================

- 纯HTML/CSS/JavaScript单文件（index.html约8500行）
- 语音听写：科大讯飞(iFlytek) WebSocket API，HMAC-SHA256鉴权
- 语音回退：FunASR公共WebSocket(wss://www.funasr.com:10095/)
- 最终回退：Google SpeechRecognition(webkitSpeechRecognition)
- 数据存储：浏览器localStorage（私人语音库、自定义词典）
- 音频处理：Web Audio API + ScriptProcessorNode（PCM 16kHz）
- 匹配算法：LCS最长公共子序列 + 拼音近似 + 别名映射
- 部署：GitHub Pages

========================================
四、硬约束（不可违反）
========================================

1. 语音识别必须用国内服务，Google SpeechRecognition仅作为最终兜底
2. 语音识别必须支持：普通话、莆仙话、闽南话
3. iFlytek API每日500次免费调用限制，超限后必须自动回退
4. 语音服务切换必须通过配置完成，不能修改核心逻辑
5. 三层回退逻辑：iFlytek → FunASR → Google SpeechRecognition
6. GitHub Pages部署，main分支根目录
7. 所有数据存在localStorage，不上传服务器
8. 单文件架构，所有代码在index.html中

========================================
五、关键配置
========================================

科大讯飞API配置（需要替换为自己的凭据）：
```javascript
const XUNFEI_CONFIG = {
  appId: '你的APPID',
  apiKey: '你的APIKey',
  apiSecret: '你的APISecret'
};
```

获取方式：访问https://www.xfyun.cn/ → 注册账号 → 创建应用 → 开通"语音听写"服务

方言参数设置：
- 中文普通话：domain='iat', accent='mandarin'
- 莆仙话/闽南话：domain='xfime-mianqie'（方言免切模式）

========================================
六、状态变量规范（极易出错，务必遵守）
========================================

项目中有两组完全独立的语音识别状态，绝对不能混用：

麦克风按钮（语音选台）：
- 状态变量：_isRecognizing
- Session变量：funasrSession
- 清理函数：stopFunasrRecognition()

录入按钮（保存到私人语音库）：
- 状态变量：_saveRecognizing
- Session变量：_saveSession
- 清理函数：stopSaveRecognition()

修改语音识别代码时，必须先确认自己在改哪一组状态。

========================================
八、开发规范
========================================

1. 每次修改index.html后，必须同步到ama-channel.html：
   cp index.html ama-channel.html

2. 提交代码：
   git add -A
   git commit -m "修改说明"
   git push

3. 推送后等1-2分钟，GitHub Pages才会更新

4. 测试时必须强制刷新：Cmd+Shift+R（Mac）或Ctrl+F5（Windows）

5. 移动端适配要求：
   - 搜索栏使用flex-wrap: wrap自动换行
   - 按钮最小触控区域44px
   - 字体加大、高对比度

========================================
九、匹配逻辑
========================================

搜索优先级（从高到低）：
1. 私人语音库匹配（最高优先级）
2. 清洗后的翻译文本（去除"我要看""帮我找"等口语）
3. 翻译文本（方言词典翻译后的中文）
4. 清洗后的原始文本
5. 原始文本
6. 拼音近似匹配

匹配结果处理：
- 只有1个节目匹配：弹出节目卡片预览1.5秒，然后自动播放
- 多个节目匹配：弹出卡片，显示"就是这个"和"换一个"按钮供选择
- 文字搜索（打字）：始终弹出确认卡片

最低分数门槛：15分（低于此分数视为无匹配）

========================================
十、已知踩坑记录
========================================

1. FunASR识别能力差、速度慢 → 已替换为iFlytek主服务
2. iFlytek报错11200（额度超限）→ 已实现三层回退
3. 私人语音库按钮无法点击 → JS必须在HTML弹窗之后执行（放在</body>前）
4. 搜索栏按钮手机端溢出 → 移动端改为flex-wrap: wrap
5. micBtn双事件监听器冲突 → 移除了initSpeechRecognition中的降级监听器
6. 语音识别状态冲突 → 创建独立的状态变量和清理函数
7. 浏览器缓存不更新 → 强制刷新Cmd+Shift+R
8. GitHub push 403错误 → Personal Access Token需要有repo权限
